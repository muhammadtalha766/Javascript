Synchronous:
The JavaScript engine executes our program sequentially, one line at a time from top to bottom in the exact order of the statements.
Synchronous is a blocking architecture, so the execution of each operation depends on completing the one before it.
Asynchronous:
Asynchronous is a non-blocking architecture, so the execution of one task isn't dependent on another.
Callbacks:
A callback is function pased as an argument to another function.
Callback Hell:
Nexted callbacks stacked below one another forming a pyramid structure.
(It is also called pyramid of Doom)
this style of programming becomes difficult to understand and manage.
Promises
Promises is for "eventual" completion of task. It is an Object in Js
It is a solution to callback hell.
 let promises = new Promises((resolve, reject) => {....})
*resolve and reject are callbacks provided by JS
Async-Await
Async function always returns a promise.
async function myFunc(){....}
await pauses the execution of its surrounding async function until the promise is settled.
